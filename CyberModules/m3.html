M3: Vulnerability Management

Pre-Class Activity

Consider the third-party or pre-existing codebases, libraries, or frameworks you are using or will use to develop your software solution. What is your plan to ensure that these software components are secure?

Video

Outline

  • A case study of the Equifax data breach
  • The broader picture of security management
  • Security issues related to open-source components
  • Source code analysis tools and software component analysis tools
  • The use of the Common Vulnerabilities and Exposures (CVE) database, the National Vulnerability Database (NVD), and the Common Vulnerability Scoring System (CVSS)
  • The inventory and monitor components of vulnerability management.

Hands-On Activity

Create an up-to-date inventory of the third-party or pre-existing codebases, libraries, or frameworks that you are using or will use. Develop a plan to review their potential vulnerabilities.

Application

  1. Discuss how this module can be applied to your project.
  2. List tasks relevant to your project based on consideration of this module.
